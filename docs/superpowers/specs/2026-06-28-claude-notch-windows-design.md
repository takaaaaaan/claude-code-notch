# Claude Notch for Windows — 設計仕様書

- **日付**: 2026-06-28
- **ステータス**: 設計確定（実装計画前）
- **目的レベル**: 本気で配布（一般ユーザー向けインストーラ配布を想定）

## 1. 概要

Claude Code 専用の Windows 版 Dynamic Island（ノッチ風オーバーレイ）。
画面の端に普段は隠れていて、**マウスを当てると降りてくる**。Claude Code の Hooks が
発するイベントをローカルで受信し、**重要イベントは通知カード**、**軽いイベントは
キャラクターの絵の変化**で知らせる。

v1 は **表示専用**（ノッチからの操作・返信は行わない）。

### コンセプトの肝
- 常駐するが普段は邪魔をしない（隠れている）
- Claude Code が「今なにをしているか」がキャラの変化で一目でわかる
- 「終わったよ」「応答して」の重要な瞬間だけカードが出る
- 配布時の最大の関門である **Hooks 設定をアプリ側が手厚く補助する**

## 2. スコープ

### v1 に含む
- ローカル HTTP サーバーによる Hooks イベント受信
- ノッチ Window（透過・枠なし・最前面）
- 5 イベントの表示（下記）
- ホバー表示 + 通知時の自動表示（約2〜3秒）
- 複数 Claude Code セッションの統合表示
- 設定 Window（6 タブ、下記）
- トレイ常駐
- Hooks セットアップ補助（スニペット生成・ワンクリック登録・登録判定）

### v1 に含まない（将来拡張）
- ノッチからの操作・返信（許可待ちへの Yes/No、ターミナルへのジャンプ）
- メディア操作 / ファイル一時置き場 / システム情報（汎用ノッチ機能）
- 複数セッションのキュー件数バッジ・一覧展開
- Mac / Linux 対応

## 3. 技術スタック

- **Electron**（Main プロセス + Renderer 複数 Window）
  - 決定理由: モックアップの HTML/CSS をそのまま活かせ、最速で形にできる。
    Claude/ChatGPT(Win)/Slack/VS Code 等、AIチャット・生産性系デスクトップアプリの
    事実上の標準。透過・最前面・トレイ・グローバル監視の実装例が豊富。
  - トレードオフ: バイナリ100MB超・常駐メモリ大。許容する。
    （将来、軽量化が課題化したら Tauri 移行を検討の余地あり）
- ローカルサーバー: Node 標準 `http` モジュール（Main プロセス内）
- フロント: 素の HTML/CSS/JS（モックアップ流用）。フレームワークは v1 では不使用

## 4. アーキテクチャ

```
┌─────────────────┐   POST localhost:<port>/event   ┌──────────────────────┐
│  Claude Code    │ ──────────────────────────────► │  Electron アプリ       │
│  (Hooksスクリプト)│   {event, session, payload}     │                       │
└─────────────────┘                                 │  ┌────────────────┐   │
                                                     │  │ ローカルHTTP    │   │
   Hooks → イベント:                                  │  │ サーバー(main)  │   │
   - Stop          → タスク完了カード                 │  └───────┬────────┘   │
   - Notification  → 許可待ちカード                   │          │ IPC         │
   - PreToolUse    → キャラ:作業中                    │  ┌───────▼────────┐   │
   - PostToolUse   → キャラ:待機へ戻す                │  │ ノッチWindow    │   │
   - SubagentStop  → キャラ:サブ完了                  │  │ (透過/最前面)    │   │
                                                     │  └────────────────┘   │
                                                     │  ┌────────────────┐   │
                                                     │  │ 設定Window      │   │
                                                     │  └────────────────┘   │
                                                     │  ┌────────────────┐   │
                                                     │  │ トレイ常駐       │   │
                                                     │  └────────────────┘   │
                                                     └──────────────────────┘
```

### コンポーネント（責務分離）

| コンポーネント | 責務 | 依存 |
|---|---|---|
| **Main / EventServer** | localhost HTTP サーバー。POST受信→検証→正規化したイベントを発行 | Node http |
| **Main / WindowManager** | ノッチ・設定 Window の生成と位置・表示制御 | Electron BrowserWindow, screen |
| **Main / HoverWatcher** | カーソル座標を監視し、トリガー帯への進入/離脱を判定 | Electron screen, グローバルタイマー |
| **Main / TrayController** | トレイアイコン・メニュー・接続状態表示 | Electron Tray |
| **Main / SettingsStore** | 設定の読み書き（JSON 永続化）と変更通知 | ファイルシステム |
| **Main / HooksInstaller** | `~/.claude/settings.json` の読取・スニペット生成・追記・登録判定 | ファイルシステム |
| **Renderer / Notch UI** | イベントを受けてカード展開/キャラ変化を描画。表示状態は持つが受信はしない | IPC |
| **Renderer / Settings UI** | 設定の表示・編集。変更を Main に送る | IPC |
| **hooks/notify script** | Claude Code の Hooks から呼ばれ、stdin JSON を localhost に POST | Node もしくは curl |

各コンポーネントは IPC / イベント発行で疎結合。EventServer は UI を知らず、
正規化イベントを出すだけ。Notch UI は通信を知らず、与えられた状態を描くだけ。

## 5. イベントとマッピング

Hooks で取得し、以下のように振り分ける。

| Hook イベント | ノッチの反応 | 表示 |
|---|---|---|
| `Stop` | タスク完了カード（緑バッジ） | カードを自動表示 → 約2〜3秒 → 収納 |
| `Notification` | 許可待ちカード（黄バッジ） | カードを自動表示 → 約2〜3秒（設定で延長可） |
| `PreToolUse` | キャラ「作業中」（青・パルス） | キャラのみ変化（カードは出さない） |
| `PostToolUse` | キャラ「待機」に戻す | キャラのみ変化 |
| `SubagentStop` | キャラ「サブ完了」（紫・バウンド）→ 待機へ | キャラのみ変化 |

### 受信ペイロード（正規化後）

```json
{
  "event": "Stop | Notification | PreToolUse | PostToolUse | SubagentStop",
  "sessionId": "string",
  "project": "cwd の末尾ディレクトリ名",
  "message": "表示用テキスト（任意）",
  "tool": "ツール名（PreToolUse 等で任意）",
  "ts": 0
}
```

Hooks スクリプトが Claude Code の hook 入力（`hook_event_name`, `session_id`,
`cwd`, `tool_name` 等）から上記へ変換して POST する。

## 6. 表示ロジック

- **通常時**: ノッチは隠れ、画面端に極薄のトリガー帯（1〜2px相当の検知領域）のみ
- **ホバー時**: カーソルがトリガー帯に入る → スッと展開して表示、離れると収納
- **通知時**: 自動展開 → 規定秒数（既定 約2〜3秒）→ 自動収納。
  - ホバー中・カーソルがノッチ上にある間は収納しない
  - イベント別に表示秒数を個別設定可能（許可待ちだけ長く 等）
- **複数セッション**: 1 つのノッチに順番に表示。カードに `project`（cwd 末尾）を表示して
  区別。短時間に複数届いた場合は順次表示（v1 はキュー件数バッジなし、将来拡張）

## 7. 表示位置（位置微調整）

プリセット + 微調整 + モニター選択。

### 位置プリセット
- **上中央**（デフォルト。横向きピル＝ノッチらしい形）
- **上左 / 上右**
- **左端中央 / 右端中央**（縦向きピル。サイドから出る）

サイド配置時は、ノッチの向き（縦長）とホバー検知帯（左右端の中央）も
それに合わせて移動・回転する。

### 微調整
- 選択プリセットからの**オフセット**（上配置なら横方向、サイド配置なら縦方向）
- **ノッチサイズ**（小 / 中 / 大）

### マルチモニター
- **表示モニターの選択**（接続中のディスプレイから選ぶ）
- モニター構成変更時は安全な既定（メインモニタ上中央）にフォールバック

## 8. 設定ウィンドウ（6 タブ）

左サイドバーでタブ切替。

### ① 一般
- Windows 起動時に自動起動 ON/OFF
- 言語: 日本語 / English
- テーマ: ダーク / ライト / システム追従
- バージョン表示・手動アップデート確認

### ② 接続（中核）
- 待ち受けポート番号（変更可、既定値あり）
- 接続ステータス: 🟢接続中 / ⚪未接続、最後の受信時刻
- **Hooks セットアップ補助**:
  - `settings.json` 用スニペットの自動生成 + コピーボタン
  - **ワンクリック登録**（`~/.claude/settings.json` を読み、Hooks を安全に追記）
  - 登録判定: ✅登録済み / ⚠未登録 を自動表示
- セキュリティ: localhost のみ受信（固定明示）。任意で簡易トークン認証

### ③ 通知
- 表示秒数スライダー（1〜10秒、既定 約2〜3秒）
- イベント別 ON/OFF（完了 / 許可待ち / ツール実行 / サブエージェント）
- イベント別 表示秒数の個別設定
- サウンド ON/OFF + 音色選択（任意）
- カード表示情報の切替（プロジェクト名 / 経過時間 / ツール名）
- 集中モード（Do Not Disturb）

### ④ ノッチ表示
- ホバー表示 ON/OFF（オフなら通知時のみ表示）
- ホバー感度（トリガー帯の高さ・反応速度）
- 位置プリセット（§7）+ オフセット + サイズ + 表示モニター選択
- アニメーション ON/OFF

### ⑤ キャラクター
- キャラ画像セットの選択（差し替え前提。v1 は仮のデフォルト）
- 各状態（待機 / 作業中 / サブ / 完了 / 許可待ち）への絵の割り当て
- キャラ表示そのものの ON/OFF

### ⑥ 詳細・デバッグ
- **テスト通知を送る**（各イベントをシミュレートして見た目を確認）
- ログ表示・エクスポート
- 設定ファイルの場所を開く / 設定をリセット

## 9. データ永続化

- 設定は JSON ファイルで保存（Electron の userData ディレクトリ配下）
- 起動時に読込、変更時に保存し UI へ反映
- スキーマにバージョン番号を持たせ、将来のマイグレーションに備える

## 10. エラー処理

- ポート使用中 → 別ポートへ自動フォールバック、設定で変更可、トレイ/設定に通知
- 不正・不明な POST → 無視（デバッグログのみ記録）
- Claude Code 未接続 → アプリは常駐継続、トレイ/設定に「未接続」表示
- モニター構成変更 → 安全既定へフォールバック
- 設定ファイル破損 → 既定値で起動し、破損ファイルを退避

## 11. テスト方針

- **EventServer 単体**: 疑似 POST を送り、正規化イベントが正しく発行されるか
- **マッピング単体**: 各イベント → 期待する UI 状態（カード種別 / キャラ状態）
- **HoverWatcher 単体**: 座標入力に対するトリガー帯の進入/離脱判定
- **HooksInstaller 単体**: settings.json への追記・登録判定（既存設定を壊さないこと）
- **E2E**: 実際に Claude Code の Hooks を設定し、各イベントでノッチが反応するか
- 設定の「テスト通知」機能を E2E 確認の主手段に使う

## 12. 配布

- Electron Builder 等で Windows インストーラ（NSIS）を生成
- 自動起動はインストーラまたは設定からレジストリ/スタートアップ登録
